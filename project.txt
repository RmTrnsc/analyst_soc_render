Projet de Fin de Formation : SOC/SIEM avec Scénario CTFVue d'ensemble du projetCe projet simule un environnement SOC réaliste où des équipe défensive (Blue Team) surveille et analyse les activités sur une infrastructure pendant qu'une équipe offensive (Red Team) tente de capturer des flags sur une machine cible vulnérable. Architecture de la solutionComposants principauxStack de sécurité défensive :•	Wazuh : SIEM principal, collecte de logs, détection d'intrusions•	Splunk : Analyse et corrélation avancée des logs•	TheHive : Gestion des incidents et cas de sécurité (à tester)•	Cortex : Automatisation de l'analyse des observables (à tester)•	MISP : Partage de threat intelligence (à tester)Infrastructure offensive :•	Machine cible CTF (vulnérable) avec flags cachés•	Génération d'activités malveillantes variéesGestionnaire versions :•	Gitlab (utilisation possible pour documentations) Planning détaillé sur 5 jours   Installation et configuration de base•	Installation de Wazuh (Manager + Agent)•	Configuration de Splunk (instance standalone)•	Déploiement de TheHive + Cortex•	Installation de MISP•	Intégration Wazuh → Splunk (forwarding des logs)•	Configuration de l'API TheHive•	Connexion Cortex à TheHive•	Configuration initiale de MISP (feeds de threat intel)Livrables :•	Toutes les plateformes opérationnelles•	Diagramme d'architecture réseau•	Documentation des credentials et accès  Préparation de la machine cible CTFCréation de la machine vulnérable :•	Déploiement d'une VM Ubuntu/Debian•	Installation de services vulnérables : o	SSH avec authentification faibleo	Service web (Apache/Nginx) avec vulnérabilités (ex: LFI, SQLi)o	Service FTP mal configuréo	Application web custom avec failles (XSS, IDOR, etc.)Intégration des flags : (cf document exemple)•	Déploiement de l'agent Wazuh sur la machine cible•	Configuration des règles de collecte (logs système, web, SSH, etc.)•	Test de transmission des logs vers Wazuh et Splunk•	Création de dashboards Splunk de base pour la supervisionLivrables :•	Machine CTF opérationnelle avec 5 flags•	Logs remontant correctement dans le SIEM Configuration avancée du SOCWazuh - Règles de détection :•	Détection de brute force SSH•	Détection de web scanning (Nikto, dirb, gobuster)•	Détection d'exploitation SQLi•	Détection de reverse shell•	Détection d'escalade de privilègesSplunk - Dashboards et alertes :•	Dashboard "Vue d'ensemble de la sécurité"•	Dashboard "Activités suspectes"TheHive - Templates et workflows : (à tester)•	Template d'incident "Tentative d'intrusion"•	Template "Compromission détectée"•	Template "Activité post-exploitation"•	Workflow de réponse à incidentCortex - Analyseurs : (à tester)•	Configuration des analyseurs VirusTotal, AbuseIPDBMISP - Intelligence : (à tester)•	Import de feeds d'IoC•	Création d'événements pour le CTF•	Intégration MISP → TheHive  Scénarios d'attaque et testsCréation de scénarios d'attaque progressifs :Scénario 1 - Reconnaissance :•	Scan de ports (nmap)•	Énumération de services•	Web directory bruteforcingScénario 2 - Initial Access :•	Brute force SSH•	Exploitation de vulnérabilité web•	Obtention du Flag 1 et Flag 2Scénario 3 - Post-exploitation :•	Reverse shell•	Énumération interne•	Lateral movement•	Flag 3Scénario 4 - Persistence & Privilege Escalation  :•	Escalade de privilèges•	Création de backdoor•	Flag 4 et Flag 5Tests de détection :•	Exécution des scénarios d'attaque•	Vérification de la génération d'alertes•	Ajustement des règles (réduction faux positifs)•	Test du workflow complet : Détection → Alerte → Incident TheHive → Analyse Cortex → Enrichissement MISPLivrables :•	Scénarios d'attaque documentés•	Guide de capture des flags  Exercice pratique et présentationExercice en conditions réellesOrganisation :•	Blue Team (défense) : 2-3 personnes o	Surveillance des dashboards Splunk/Wazuho	Analyse des alerteso	Création de cas dans TheHiveo	Enrichissement avec Cortex/MISP•	Red Team (attaque) : 2-3 personnes o	Tentative de capture des 5 flagso	Essai d'être le plus furtif possibleo	Documentation des techniques utiliséesPrésentation et documentationPrésentation du projet :•	Architecture de la solution SOC•	Démo des outils (Wazuh, Splunk, cores•	RAM : 32 GB•	Stockage : 500 GB SSD•	OS : Ubuntu 22.04 LTSMachine cible CTF :•	CPU : 2 cores•	RAM : 4 GB•	Stockage : 40 GB•	OS : Ubuntu 20.04 LTSMachines attaquants (Red Team) :•	Kali Linux ou Parrot OS•	CPU : 2 cores•	RAM : 4 GBRéseau[Internet] ← NAT Gateway    |    ├── [SOC Network - 192.168.10.0/24]    │   ├── Wazuh Manager:     │   ├── Splunk:     │   ├── TheHive: Automatisation : Utilisez des scripts pour le déploiement rapide2.	Snapshots : Prenez des snapshots VM après chaque étape majeure3.	Documentation continue : Documentez au fur et à mesure, pas à la fin4.	Tests réguliers : Testez chaque composant avant de passer au suivant5.	Travail d'équipe : Divisez les tâches selon les compétences6.	Backup : Sauvegardez régulièrement les configurationsBon courage pour votre projet ! 🛡️
